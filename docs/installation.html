Installation Guide

Get MergeAssistant up and running on your system in minutes

This guide outlines the necessary steps to install and run MergeAssistant. The application uses an automated setup that handles Python, virtual environments, and dependencies for you.

Prerequisites

RequirementDetails
Operating SystemmacOS 10.15+, Windows 10/11, or Linux (Ubuntu 20.04+)
Disk Space~500MB for application and dependencies
Memory4GB RAM minimum (8GB recommended for large packages)
NetworkInternet connection required for first-time setup
GitRequired to clone the repository
Windows-Specific: The Windows application may require .NET Framework 4.6.2 or later. Most Windows 10/11 systems have this pre-installed. If you encounter startup errors, install .NET Framework 4.8.

Installation Steps

1 Clone the Repository

Open a terminal (macOS/Linux) or Command Prompt (Windows) and run:

bash
git clone git@github.com:ram-020998/3WM.git

2 Navigate to the Project Directory

Change your current directory to the newly cloned project folder:

bash
cd 3WM

3 Make Scripts Executable (macOS/Linux Only)

On macOS and Linux, ensure the startup scripts are executable. Windows users should skip this step.

bash
chmod +x 3wm.sh scripts/3wm.sh

4 Install Global Command

To run 3wm from any directory without navigating to the project folder:

PlatformCommand
macOS / Linux./3wm.sh install-cli
Windows3wm.bat install-cli

5 Start the Application

The application uses an automated setup script that handles Python, virtual environments, and dependency installation on the first run.

Operating SystemCommand
macOS / Linux3wm start
Windows3wm start

What Happens During First Run

The startup script will perform the following automated setup steps:

StepDescriptionTime (Approx.)
1Downloads uv package manager (~15MB)~5 seconds
2Installs Python 3.11 (if not present)~10 seconds
3Creates isolated virtual environment~2 seconds
4Installs all application dependencies~15 seconds
5Starts the Flask web server~3 seconds

Total First-Run Time: 30-45 seconds

The uv package manager and Python installation are stored locally in the .uv/ directory within the project folder, ensuring complete isolation from your system's global Python installations.

Stopping the Application

To stop the running application, go back to the terminal window where the server is running and press:

Operating SystemKey Combination
macOS / LinuxCtrl + C
WindowsCtrl + C

Troubleshooting

ProblemSolution
"Permission denied" (macOS/Linux)Run chmod +x 3wm.sh scripts/3wm.sh then try again.
"Port 5000 already in use"Stop the application currently using port 5000, or modify the port in app.py.
".NET Framework" error (Windows)Install .NET Framework 4.8.
Application crashes (large packages)Close other memory-intensive programs, ensure 8GB RAM is available.

Database Errors (SQLite)

If you encounter SQLite database corruption or lock errors:

bash
# Remove the database and restart (data will be lost) 3wm reset-db

Uninstalling

To completely remove the application and all associated data:

bash
3wm uninstall

Setting up AI Summary

Configure native AI summarization using Appian's Summarizer AI agent

This section details the steps required to set up native AI summarization for the MergeAssistant application. This approach utilizes an Appian application package that includes a Summarizer AI agent exposed as a Web API service.

Prerequisites

1 Download the Appian Application Package

Obtain the application package from the following Google Drive link:

Download from Google Drive

2 Import the Application

Import the downloaded Appian application package into your target environment.

Generating the API Key for MergeAssist Communication

Once the application is imported, follow these steps to generate the required API key for MergeAssist to communicate with the Summarizer AI agent:

1 Sign in as Admin

In the environment where you have deployed the application, sign in as the admin user.

2 Navigate to Web API Authentication

Navigate to the Admin Console and select Web API Authentication.

3 Create New API Key

From the API Key tab, click CREATE.

4 Add Description

In the Description field, add a unique description for the API key:

text
MergeAssist_AISummary_Key

5 Create Service Account

Click the blue circle Plus icon next to the Service Account field to create a new service account.

FieldValue
Usernamemergeassistant.ai.service
GroupsTWM Administrators

Click CREATE to apply the new service account to the API key.

6 Finalize and Save API Key

Click CREATE to finalize the new API key.

Important: Copy and securely save the generated API key. You will need it in the next section. This key cannot be retrieved later.

Configuring AI Summary in MergeAssistant

1 Open Settings

Navigate to the Settings page in the MergeAssistant application.

2 Configure AI Features

In the AI Features section:

SettingValue
AI Summary generationTurn ON the toggle
AI ProviderSelect Appian Merge Summary API
API KeyPaste the API key you copied in the previous steps
API End PointSee format below

3 Set API Endpoint

For the API End Point field, use the following format, replacing <your-env-url> with the base URL of your Appian environment:

url
https://<your-env-url>/suite/webapi/generateChangeSummary

4 Test the Connection

After saving the configuration, click on Test Provider to verify that the connection and configuration were successful.

A successful test will confirm that MergeAssistant can communicate with the Appian Summarizer AI agent and generate summaries for your merge changes.
